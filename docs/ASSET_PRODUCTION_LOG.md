# Asset Production Log

## 2026-07-05 - Core 16 Starting Asset Pass

Created a structured first asset package at:

```text
src/assets/core_16/
```

The package is a new side-by-side pass and does not overwrite the existing
`src/assets/tiles/layer1` or `src/assets/tiles/layer2` assets.

## What Was Produced

- 5 material tile families: dirt, stone, copper, hardstone, iron.
- 3 clean variants per material.
- 3 damage states per material: cracked 1, cracked 2, near break.
- 4 back-wall/tunnel variants for mined cells.
- 3 resource icons: stone, copper, iron.
- Mining feedback assets: hit flash, target marker, break dust, and material chips.
- Mine structure starters: entrance and elevator frame.
- Worker starter frames: idle, walk, wind-up, hit, recover.
- Camp starters: tent, upgrade board, pickaxe prop, stamina barrel, elevator crate.
- Native tilesheet and JSON frame metadata.
- Preview sheets for visual QA.
- `manifest.json` with paths, sizes, categories, and tags.

Total generated asset records: 68.

## Key Art Decisions

- Tiles are native 16x16 to test the more zoomed-out mine direction.
- Readability beats detail. Each tile uses sparse marks and strong material color.
- All breakable mine tiles share the same construction grammar: dark seam, top
  highlight, bottom shadow, and controlled internal texture.
- Ore is embedded in rock, but copper/iron tiles always mean copper/iron reward.
- Resource icons are cleaner UI silhouettes than the ore chunks inside tiles.
- Hardstone is intentionally darker and heavier so it reads as a progression gate.
- The mined state is a dark physical back-wall tile, not transparent or pure black.

## Generator

Assets are generated by:

```text
scripts/generate_core16_assets.py
```

Run from the project root:

```powershell
python scripts\generate_core16_assets.py
```

The generator is deterministic and can be rerun after palette or shape edits.
